
# Lessons Learned

## Project Information

| Item | Details |
|------|---------|
| Project Name | Sample Web Application |
| Project Manager | Ranjith Babu |
| Methodology | Agile Scrum |
| Duration | 6 Months |

---

# What Went Well

- Sprint planning was completed on schedule.
- Daily stand-ups improved team communication.
- Stakeholder engagement remained consistent throughout the project.
- Automated testing reduced production defects.

---

# Challenges Faced

- Requirement changes during Sprint 3.
- Resource availability impacted sprint velocity.
- Delayed client approvals.

---

# Lessons Learned

- Conduct backlog refinement before every sprint.
- Involve stakeholders earlier during requirement gathering.
- Improve sprint estimation using historical velocity.
- Maintain a proactive risk register.

---

# Recommendations

- Improve communication cadence.
- Increase automation coverage.
- Conduct regular knowledge-sharing sessions.

---

# Project Outcome

Project delivered successfully with high customer satisfaction.
